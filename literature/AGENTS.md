# literature/AGENTS.md -- Literature Review Management System v3

Read this file first to understand the agent-native literature review system.

## What is the Literature System?

The `literature/` directory is a **portable, agent-native literature review management system** designed to help researchers and teams systematically collect, organize, and synthesize academic papers and resources. It provides:

1. **Structured metadata** -- YAML frontmatter for papers with rich relationship graphs
2. **SQLite index** -- `papers.db` rebuilt from markdown files via `lit rebuild`; fast BM25 search, PageRank, and funnel retrieval
3. **Agent-friendly queries** -- `lit` CLI for search, recommendations, synthesis, and discovery
4. **LaTeX integration** -- Generate formatted literature reviews from paper collections

## System Architecture

**Three-layer design:**

- **Data layer** (`literature/papers/*.md`) -- Portable, version-controlled paper metadata, notes, and summaries
- **Index layer** (`papers.db`) -- SQLite database derived from markdown files; rebuilt with `lit rebuild`; delete to force full rebuild
- **Agent layer** (`lit` CLI) -- `literature/scripts/lit.py` wraps all operations; use `uv run python literature/scripts/lit.py <COMMAND>`

**Location convention:** All paths are relative to the git repository root. The system is designed to work in any git worktree or clone.

**Key invariant:** Markdown files are source of truth. `papers.db` is derived and always rebuildable. Run `lit rebuild` after any change to paper files.

## Paper Frontmatter Schema

Each paper is stored as a Markdown file with YAML frontmatter. The filename must match the `doc_id` (citekey).

**File:** `literature/papers/{citekey}.md`

**Example:** `literature/papers/vaswani2017attention.md`

```yaml
---
# === IDENTITY ===
doc_id: "vaswani2017attention"        # citekey, matches filename (no .md)
title: "Attention Is All You Need"    # always quoted
authors:                              # list, first 10 max; et_al: true if more
  - "Vaswani, A."
  - "Shazeer, N."
  - "Parmar, N."
  - "Uszkoreit, J."
  - "Jones, L."
  - "Gomez, A. N."
  - "Kaiser, Ł."
  - "Polosukhin, I."
year: 2017
venue: "NeurIPS 2017"                 # conference or journal name
doi: "10.48550/arXiv.1706.03762"      # optional
arxiv_id: "1706.03762"               # optional
s2_id: "649def34f8be52c8b66281af98ae884c09aef38b"  # Semantic Scholar paper ID, optional
url: "https://arxiv.org/abs/1706.03762"
et_al: false                          # true if >10 authors

# === RESOURCE TYPE ===
resource_type: paper                  # paper | preprint | blog | report | talk | code

# === GRAPH EDGES (typed, directed) ===
cites: []                             # papers this paper cites
# Each entry: {id: "citekey", type: "extends"}
# Relationship types: cites | extends | contradicts | uses_method | uses_dataset | surveys
cited_by: []                          # populated by rebuild_index.py — do not edit manually
related: []                           # semantic neighbors, optional

# === STATUS (per collaborator) ===
reading_status:
  global: "unread"                    # unread | skimmed | read | synthesized
  # Per-collaborator (add your username key):
  # hangyu:
  #   status: "read"
  #   date: "2026-03-12"
  # junghun:
  #   status: "synthesized"
  #   date: "2026-03-01"

# === TAXONOMY ===
tags: []                              # auto-populated from S2 fieldsOfStudy + s2FieldsOfStudy by enrich.py; add your own freely
themes: []                            # maps to files in themes/ directory
contribution_type: "method"          # method | survey | benchmark | theory | dataset

# === ENRICHMENT (populated by enrich.py) ===
citation_count: 0
influential_citation_count: 0        # from S2 influentialCitationCount, always int, default 0
tldr: ""                              # AI-generated TL;DR from Semantic Scholar
abstract: ""

# === PROVENANCE (optional, set when paper is discovered via scout.py) ===
provenance:
  method: "enrich"                    # enrich | scout_recommend | scout_search | scout_gaps
  query: ""                           # search query if applicable
  seed_papers: []                     # seed citekeys if from recommendation
  discovered_at: "2026-03-13"        # ISO date

# === SUMMARIES (written by agent via set_summary() in parse.py) ===
# DO NOT edit manually -- use set_summary() to ensure provenance is recorded
summaries:
  l4:                                 # One-liner (20-30 words), used in fast scanning
    text: ""                          # The summary sentence
    model: ""                         # LLM that generated this (e.g. "claude-opus-4-6")
    generated_at: ""                  # ISO 8601 timestamp
  l2:                                 # 3-5 key claims, used for structured synthesis
    claims: []                        # List of claim strings
    model: ""
    generated_at: ""
---

# Notes

Your notes on this paper go here. Use Markdown. Agents will read this section to understand your synthesis.

## Key Contributions

- Contribution 1
- Contribution 2

## Strengths

- Strength 1
- Strength 2

## Weaknesses

- Weakness 1
- Weakness 2

## Related Work

Links to related papers and how they differ.
```

## Resource Frontmatter Schema

Non-paper resources (blogs, talks, code, reports) use the same schema but with different `resource_type` and optional fields.

**Minimal required fields:**
- `doc_id` -- unique identifier
- `title` -- resource title
- `resource_type` -- one of: `paper`, `preprint`, `blog`, `report`, `talk`, `code`
- `url` -- where to find it

**Example:** `literature/resources/attention_blog_2017.md`

```yaml
---
doc_id: "attention_blog_2017"
title: "The Illustrated Transformer"
authors:
  - "Alammar, J."
year: 2017
resource_type: blog
url: "https://jalammar.github.io/illustrated-transformer/"
reading_status:
  global: "read"
tags: ["transformers", "visualization"]
themes: ["transformers"]
---

# Notes

Great visual explanation of the Transformer architecture.
```

## Relationship Types

All 6 relationship types for the `cites` and `related` fields:

| Type | Meaning |
|------|---------|
| `cites` | This paper directly cites the referenced paper |
| `extends` | This paper builds on or extends the referenced work |
| `contradicts` | This paper disagrees with or refutes the referenced work |
| `uses_method` | This paper applies a method from the referenced paper |
| `uses_dataset` | This paper uses a dataset from the referenced paper |
| `surveys` | This paper surveys or reviews the referenced work |

## Reading Status Values

All 4 reading status values:

| Status | Meaning |
|--------|---------|
| `unread` | Paper not yet read |
| `skimmed` | Quick skim of abstract and introduction |
| `read` | Full read with notes |
| `synthesized` | Integrated into your understanding; can explain to others |

## Index Files (Auto-Generated — DO NOT EDIT)

These files are generated by `rebuild_index.py` and should never be manually edited.

### `index/graph.yaml`

YAML representation of the citation graph. Each node is a paper with its metadata and edges.

```yaml
nodes:
  vaswani2017attention:
    title: "Attention Is All You Need"
    authors: ["Vaswani, A.", "Shazeer, N.", ...]
    year: 2017
    venue: "NeurIPS 2017"
    resource_type: paper
    reading_status:
      global: unread
      hangyu:
        status: read
        date: "2026-03-12"
    tags: ["transformers", "attention"]
    themes: ["transformers"]
    contribution_type: method
    citation_count: 50000
    cites: []
    cited_by: []
    related: []

edges:
  - from: paper_a
    to: paper_b
    type: extends
```

### `index/status.yaml`

Reading status summary across all collaborators.

```yaml
global:
  unread: [citekey1, citekey2, ...]
  skimmed: [citekey3, ...]
  read: [citekey4, ...]
  synthesized: [citekey5, ...]

hangyu:
  unread: [...]
  skimmed: [...]
  read: [...]
  synthesized: [...]

junghun:
  unread: [...]
  skimmed: [...]
  read: [...]
  synthesized: [...]
```

### `index/references.bib`

Standard BibTeX format, suitable for LaTeX `\bibliography{}` commands.

```bibtex
@article{vaswani2017attention,
  title={Attention Is All You Need},
  author={Vaswani, A. and Shazeer, N. and ...},
  journal={NeurIPS},
  year={2017},
  doi={10.48550/arXiv.1706.03762}
}
```

### `index/embeddings.yaml`

SPECTER2 embedding vectors for semantic similarity search. Each vector is 768 floats, rounded to 4 decimal places.

```yaml
model: specter_v2
dimensions: 768
vectors:
  vaswani2017attention:
    - 0.1234
    - -0.5678
    # ... 768 floats total
  devlin2019bert:
    - 0.9012
    # ...
```

Generated by `rebuild_index.py` when `S2_API_KEY` is set. Used by `query.py similar` and `landscape.py` for semantic similarity. Papers without `s2_id` are skipped. Pass `--skip-embeddings` to skip generation even when an API key is present.

## Landscape Report Files (Auto-Generated — DO NOT EDIT)

These files are generated by `landscape.py` and should never be manually edited.

### `landscape.yaml`

Research landscape analysis: clusters, research fronts, structural holes, and citation gaps.

```yaml
generated_at: "2026-03-13T12:00:00"
paper_count: 17
cluster_count: 4
clusters:
  0:
    label: ["attention", "transformer", "self-attention"]
    papers: ["vaswani2017attention", "devlin2019bert"]
    size: 2
    avg_year: 2018.5
    avg_citations: 140183
research_fronts:
  - cluster_id: 2
    label: ["diffusion", "denoising"]
    velocity: 8.5
    key_papers: ["ho2020denoising"]
structural_holes:
  - cluster_a: 0
    cluster_b: 3
    density: 0.02
citation_gaps:
  - id: "vaswani2017attention"
    cited_by_count: 3
    cited_by: ["paper_a", "paper_b", "paper_c"]
```

**`clusters`**: Papers grouped by SPECTER2 embedding similarity. Each cluster has a label (top TF-IDF terms from abstracts), list of citekeys, size, average publication year, and average citation count.

**`research_fronts`**: Top 3 clusters by citation velocity (recent citation activity). High velocity = active area.

**`structural_holes`**: Cluster pairs with inter-cluster citation density below 0.05. Low density = research opportunity bridging two areas.

**`citation_gaps`**: Papers cited by 2 or more collection papers but not yet in the collection. Sorted by `cited_by_count` descending.

### `landscape.md`

Narrative report with:
- Summary: paper count, cluster count
- Per-cluster summary: label, key papers, avg year/citations
- Research fronts: active areas by citation velocity
- Structural holes: research opportunities between clusters
- Citation gaps: missing influential papers
- Reading recommendations: front papers + gap papers

## Script Commands

All scripts are invoked via `uv run python` from the repository root.

### Primary CLI: `lit.py` (v3)

The `lit` command is the primary interface for all v3 operations. Use it instead of calling individual scripts directly.

```bash
# Rebuild SQLite index from markdown files (run after ANY changes)
uv run python literature/scripts/lit.py rebuild

# BM25 full-text search
uv run python literature/scripts/lit.py search "limit order book simulation"
uv run python literature/scripts/lit.py search "diffusion models" --top-k 10
uv run python literature/scripts/lit.py search --similar vaswani2017attention

# Show paper details and summaries
uv run python literature/scripts/lit.py paper vaswani2017attention

# Reading recommendations (N is a positional number, NOT "next N")
uv run python literature/scripts/lit.py recommend 5

# Cross-paper synthesis via funnel retrieval
uv run python literature/scripts/lit.py ask "What generative models exist for LOB data?" --depth 2

# Discover new papers
uv run python literature/scripts/lit.py discover --source s2
uv run python literature/scripts/lit.py discover --source arxiv --categories cs.LG,q-fin.TR

# View and act on discovered papers
uv run python literature/scripts/lit.py inbox
uv run python literature/scripts/lit.py inbox add <paper_id>
uv run python literature/scripts/lit.py inbox dismiss <paper_id>

# Add a paper from URL
uv run python literature/scripts/lit.py add "https://arxiv.org/abs/1706.03762"

# Summarization queue
uv run python literature/scripts/lit.py ingest --list    # papers needing summaries
uv run python literature/scripts/lit.py ingest --status  # ingestion progress

# Reading queue status
uv run python literature/scripts/lit.py status

# Collection statistics
uv run python literature/scripts/lit.py stats

# Generate LaTeX review
uv run python literature/scripts/lit.py generate --title "My Survey" --authors "Author Name"

# Migrate from v1
uv run python literature/scripts/lit.py migrate --from-v1
```

### Summarization API (Python)

After reading a paper, store summaries with provenance using the Python API:

```python
from literature.scripts.parse import read_frontmatter, write_paper_file, set_summary
from pathlib import Path

meta, body = read_frontmatter(Path("literature/papers/vaswani2017attention.md"))

# L4: one-liner summary
set_summary(meta, "l4",
    "Transformer introduces self-attention as a complete replacement for recurrence, achieving BLEU state-of-the-art while being more parallelizable.",
    "claude-opus-4-6")

# L2: key claims list
set_summary(meta, "l2",
    ["Self-attention captures long-range dependencies in O(1) operations vs O(n) for RNNs",
     "Multi-head attention attends to different representation subspaces jointly",
     "Positional encoding preserves sequence order without recurrence"],
    "claude-opus-4-6")

write_paper_file(Path("literature/papers/vaswani2017attention.md"), meta, body)
# Then run: uv run python literature/scripts/lit.py rebuild
```

## Conventions

1. **Always `git pull` before working**, `git push` after done.
2. **Run `lit rebuild` after ANY changes** to paper or resource files (v3 preferred; `rebuild_index.py` also works).
3. **Never manually edit `papers.db`** -- it is auto-generated by `lit rebuild`.
4. **Never manually edit files in `literature/index/`** -- they are auto-generated.
5. **Citekey format:** `{firstauthorlastname}{year}{titleword}` all lowercase, no spaces.
   - Example: `vaswani2017attention`, `devlin2018bert`, `brown2020language`
6. **Missing collaborator status** is treated as `unread` by the index scripts.
7. **Paper files must be valid Markdown** with YAML frontmatter.
8. **All URLs must be absolute** (start with `http://` or `https://`).
9. **Author names** should follow "LastName, FirstInitial." format.
10. **Themes** should be lowercase, hyphen-separated (e.g., `large-language-models`).
11. **Tags** are free-form but should be consistent across papers.
12. **Summaries require provenance** -- always pass your model name to `set_summary()`.
13. **Markdown files are source of truth** -- `papers.db` is derived and rebuildable from scratch by deleting it and running `lit rebuild`.

## How to Help

When working with this system:

- **Maintain metadata quality** -- accurate frontmatter enables agent queries
- **Write clear notes** -- agents will read your synthesis to understand papers
- **Keep relationships updated** -- link related papers so agents can traverse the graph
- **Run rebuild_index.py** -- after any changes, regenerate indices
- **Test agent queries** -- verify that agents can find papers by theme, tag, or relationship

## Rules

1. **Keep indices in sync.** Whenever you add, remove, or modify a paper file, run `rebuild_index.py` in the same session.
2. **Never commit generated files.** The `literature/output/` directory is gitignored. Only commit source files in `papers/`, `resources/`, and `themes/`.
3. **Preserve frontmatter structure.** The YAML frontmatter is machine-readable. Maintain the exact field names and types.
4. **Document relationships.** When you read a paper, add `cites` entries to link it to related work.
